# Backend A - Supabase Edge Functions Core

## Arquitetura Geral
- Tipo: HTTP functions (Supabase Edge Functions) com handlers Deno (`serve`) e payload JSON. (fonte: supabase/functions/discover-data-api/index.ts:594)
- Domínios: Discover (`discover-*`), DPPI (`dppi-*`), TGIS (`tgis-*`), AI Analyst (`ai-analyst`). (fonte: supabase/config.toml:3)
- Chamada cliente predominante: `supabase.functions.invoke(...)`. (fonte: src/lib/discoverDataApi.ts:61)

## Estratégia de Auth/AuthZ
- `verify_jwt` é definido por funçăo em `supabase/config.toml`. (fonte: supabase/config.toml:3)
- `discover-data-api` aplica ACL por operaçăo/tabela e role (`public|authenticated|admin`). (fonte: supabase/functions/discover-data-api/index.ts:219, supabase/functions/discover-data-api/index.ts:497)
- `x-doc-status: incomplete` para funçőes sem contrato formal explícito (schema request/response năo tipado em OpenAPI no código).
